    (inner, constant) docsubject

    docsubject is the "global" subject which keeps track of if the spawned child processes have completed or not.

    When they are all completed we can start to generate the index-page of the documentation.

    (inner, constant) repo$

    Kick it. Uses the global subject in each spawned subprocess to capture the completion of the subprocess so we do not start to process the index generation until all subprocesses have completed.
